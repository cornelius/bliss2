# bliss show — reference mockup
#
# Focus mode: same scoping as bliss list, filtered for action.
# Key differences from bliss list:
#   - No header line, ever.
#   - Incoming omitted unless non-empty.
#   - Filtered view (bliss show <list>): no list name, no indent.
#   - Empty state: "All done. Nothing left to do." (stMuted)
#
# Unfiltered items: "%3d  %s\n" (right-aligned number in 3 chars, 2 spaces, title)
# Filtered items:   "%d  %s\n"  (no padding, numbers start at column 0)
# Section delimiter: listSectionDelim() helper (same as bliss list)
# List name: stBold, no indent. Blank line between lists.
